# 🧟 SQL-аналитика мобильной игры Zombie Revolution

Продуктовая и финансовая аналитика мобильной игры на **PostgreSQL**: активность игроков, вовлечённость, выручка, когорты, вирусность и качество данных.

> Учебный проект (курсовая SkyPro), оформленный как портфолио-кейс.

---

## 📌 О проекте

**Zombie Revolution** — мобильная игра. На основе сырых данных об игровых сессиях, пользователях, покупках и реферальных приглашениях нужно было ответить на продуктовые и финансовые вопросы бизнеса и подготовить презентацию с выводами.

**Период данных:** 01.07.2022 – 30.04.2023
**Объём:** 3 078 пользователей, 27 161 игровая сессия.

### Стек
- **PostgreSQL** — все расчёты: `JOIN` (в т.ч. join по неравенству), `GROUP BY`/`HAVING`, `CASE WHEN`, подзапросы, `date_trunc`, `COALESCE`, работа с интервалами.
- **Excel** — постобработка выгрузок и визуализация (объединённые графики DAU/WAU/MAU, Sticky Factor).

## 🗂 Структура данных (схема `skygame`)

## ❓ Задачи и ключевые выводы

### 1. Маркетинговые метрики: DAU / WAU / MAU + Sticky Factor
Оценить эффект акции (первые 3 недели марта 2023, раздача бесплатных кристаллов за частые заходы).
- 📈 **Акция сработала на активность:** число заходов выросло на **≈70–80%**.
- ⚠️ **Но Sticky Factor упал:** новые игроки не стали заходить *чаще* — привлекли аудиторию, но не удержали привычку. Рост активности — за счёт объёма, а не вовлечённости.
- 🚀 С января 2023 прирост базы удвоился, рекордный месяц — **март 2023**.

📄 [`queries/01_marketing_metrics.sql`](queries/01_marketing_metrics.sql)

### 2. Качество данных (Data Quality)
У части сессий не заполнено `end_session`.
- **3 262** проблемных сессии — **12%** всех строк.
- **97.61%** всех проблемных записей приходится на **iOS** (у iOS проблемные — 19% его сессий против 0.75% у Android). Явный баг трекинга на iOS.

📄 [`queries/02_data_quality.sql`](queries/02_data_quality.sql)

### 3. Самые вовлечённые игроки (топ-25 по времени в игре)
- Лидер — игрок `2902744`: **6 дней 18 часов 33 минуты** суммарно.
- Средняя длительность сессии ~**2.4 ч**, в марте пик до **5.54 ч**.

📄 [`queries/03_engaged_players.sql`](queries/03_engaged_players.sql)

### 4. Выручка по типам товаров
- Лидеры продаж: **Ammo** (боеприпасы), **Currency** (валюта), **Materials** (материалы).
- Расчёт по цене, **актуальной на момент покупки** (двойной join по неравенству к истории цен `log_prices` + `COALESCE` для открытого интервала).

📄 [`queries/04_revenue.sql`](queries/04_revenue.sql)

### 5. Повышение цены на кристаллы (с 01.01.2023) и когортная выручка
- После подорожания **количество** проданных кристаллов упало, а **суммарная выручка выросла** — спрос неэластичен, повышение цены оправдано.
- Средняя месячная выручка на клиента по когортам растёт.

📄 [`queries/05_cohorts_revenue.sql`](queries/05_cohorts_revenue.sql)

### 6. Лояльность когорт и вирусность (K-factor)
- Гипотеза подтвердилась: когорта **ноя–дек 2022** (дорогая таргетированная реклама) играет дольше — **217 мин** против **177 мин** у остальных (**+40 мин/сессия**).
- **K-factor = 0.92** — каждый игрок приводит в среднем 0.92 нового. Прогнозируемый размер будущей когорты ~**314** игроков.

📄 [`queries/06_loyalty_kfactor.sql`](queries/06_loyalty_kfactor.sql)
